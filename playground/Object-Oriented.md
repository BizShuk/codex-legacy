# OO principles
ref: http://wiki.c2.com/?PrinciplesOfObjectOrientedDesign

### Principle of Class
- Single Responsibility Principle (SRP)
    Only do one thing.

- Open-Closed Principle (OCP)
    Open for extension, but close for modification

- Liskov Substitution Principle (LSP)
    If all objects are belonging one type of class (parent or interface), all objects can be substituded without knowing what object it really is. For example, Square and Rectagle

- Dependency Inversion Principle (DIP)
    Abstraction should not depend on low level module

- Interface Segregation Principle (ISP)
    Segregate unnecessary interface

### Principle of Cohesiono
- Reuse Release Equivalence Principle (REP)
- Common Closure Principle (CCP)
- Common Reuse Principle (CRP)

### Principle of Coupling
- Acyclic Dependency Principle (ADP)
- Stable Dependency Principle (SDP)
- Stable Abstraction Principle (SAP)
